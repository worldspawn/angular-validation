Validators

Below is a list of validators with examples of use.

All validators support providing a function as the rule comparison value. The function will get the object being validated as its only argument. function (obj) {}

If the comparison value is a Date then Assent will attempt to convert the model value to a Date, if it fails to convert the validator will pass

notEmpty

notEmpty can target text fields and array fields. It will fail for null, undefined or empty string values and zero length arrays.

c.ruleFor('username', function (f) {
  f.notEmpty()
    .withMessage('username is required');
});

email

email can target text fields. It performs a basic check for a email@domain style value.

c.ruleFor('email', function (f) {
  f.email()
    .withMessage('must be a valid email');
});

minLength

minLength can target text fields and array fields. It will fail for string values that are shorter than the specified minimum length or arrays that contain less items that the specified minimum length.

c.ruleFor('password', function (f) {
  f.minLength(20)
    .withMessage('password must be at least {|} characters long');
});

The message is automatically provided the specified minimum length as an interpolation argument for the message.

maxLength

maxLength can target text fields and array fields. It will fail for string values that are longer than the specified maximum length or arrays that contain more items that the specified maximum length.

c.ruleFor('firstName', function (f) {
  f.maxLength(50)
    .withMessage('firstName must be at most {|} characters long');
});

The message is automatically provided the specified maximum length as an interpolation argument for the message.

min

min can target number fields or array fields. It will fail for numeric values that are less than the specified minimum or arrays that contain less items that the specified minimum.

c.ruleFor('age', function (f) {
  f.min(18)
    .withMessage('You must be at least {|} years old.');
});

The message is automatically provided the specified minimum as an interpolation argument for the message.

max

max can target number fields or array fields. It will fail for numeric values that are more than the specified maximum or arrays that contain more items that the specified minimum.

c.ruleFor('age', function (f) {
  f.max(60)
    .withMessage('You must be no more than {|} years old to ride the gravitron.');
});

The message is automatically provided the specified maximum as an interpolation argument for the message.

matches

matches can target value fields. It will fail when the field contains a value (not null) and it does not match the provided comparison value.

c.ruleFor('password', function (f) {
  f.matches('secretpassword')
    .withMessage('Your password must be \'secret password\'.');
});

A more useful scenario is to compare to the value of another field

c.ruleFor('passwordComparison', function (f) {
  f.matches(function (obj) { return obj.password; })
    .withMessage('Confirm password must match the password');
});
